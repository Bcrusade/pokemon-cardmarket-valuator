# Security Policy

## Supported Versions

This project is in early stage. Security support applies to the latest version on main branch.

## Reporting a Vulnerability

If you discover a security issue, please do not open a public issue.

Instead:

- send a private report to: your@email.com
- include a clear description
- include reproduction steps if possible

## Scope

Security issues include:

- vulnerabilities in backend endpoints
- unsafe handling of user uploads
- injection risks (HTML parsing, scraping)
- API key exposure
- authentication flaws (future features)

## Non-Security Issues

The following are NOT security issues:

- pricing inaccuracies
- card misidentification
- UI/UX bugs

Please open a standard issue for those.

## Response Policy

- we aim to acknowledge within 72 hours
- fixes will be prioritized based on severity
